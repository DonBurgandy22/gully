## Memory flush: 2026-04-17 16:48
### Cognitive framework 3D visualizer edge rendering fix
- **File:** `C:\Burgandy\burgandy-cognitive-framework\outputs\burgandy_network_3d.html`
- **Symptom:** Edges/links rendered as black, no visible color
- **Root cause:** `THREE.LineBasicMaterial` with `emissive` properties is ineffective without scene lighting. The scene has no lights (`scene.light` not initialized), so emissive rendering is invisible.
- **Material class:** `THREE.LineBasicMaterial` for graph edges, `THREE.MeshBasicMaterial` for pulse spheres
- **Invalid assumption:** Emissive color on lines requires light illumination; no scene lights exist
- **Patch applied:** Removed `emissiveIntensity` from `THREE.LineBasicMaterial` constructor. Replaced with `THREE.AdditiveBlending`, `depthWrite: false`, and increased `linewidth`/`opacity` for visible color rendering without lights.
- **Exact snippet before:** `THREE.LineBasicMaterial({ color: baseColor, emissive: baseColor, emissiveIntensity: intensity, opacity: 0.3 })`
- **Exact snippet after:** `THREE.LineBasicMaterial({ color: baseColor, transparent: true, opacity: opacity, blending: THREE.AdditiveBlending, depthWrite: false })`
- **Implementation-proven:** Yes - `AdditiveBlending` creates visible additive color rendering without scene lights. `depthWrite: false` prevents z-fighting. Bright colors (green/blue) now visible.
- **Next visual check:** Refresh browser at `http://localhost:8765/burgandy_network_3d.html` to verify edges render in bright colors (green for adaptive edges, blue for base edges) instead of black.
