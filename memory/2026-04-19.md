# 2026-04-19 Tool Calling Fix

## What broke
- models.json baseUrl had /v1 suffix breaking native Ollama tool calling
- Local models missing compat.supportsTools: true
- AGENTS.md had wrong tool names (read_file/write_file/run_command)
- MEMORY.md referenced stale model qwen3.5:4b
- Session history grew to 175 messages causing context overflow

## What fixed it
- Removed /v1 from Ollama baseUrl in models.json
- Added compat.supportsTools: true to qwen2.5 and qwen3.5:4b
- Fixed tool names in AGENTS.md to read/write/exec
- Switched to deepseek/deepseek-chat to break out of overflow
- Cleared session file to reset context

## Rules going forward
- qwen2.5:7b-instruct works for single-step exec commands
- Multi-step or complex tasks: use deepseek/deepseek-chat
- If tool calling breaks: clear session first, then check baseUrl and supportsTools
- Never let session exceed 100 messages without a manual clear
